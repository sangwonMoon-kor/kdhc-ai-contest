02

프로젝트 소개

무엇을 만드나 일정·문서·기준·결과물을 한 업무 단위로 연결하고, 수행 중 메모까지 다음 업무에 재사용하는 AI 업무 도우미입니다.

누구를 위해 한글·엑셀·전자결재 문서를 반복해서 찾고 재작성하는 유지보수 직원과 사무직원을 위한 서비스입니다.

왜 공유폴더·개인 폴더·전자결재에 흩어진 자료 때문에 준비 시간이 길고, 담당자가 바뀔 때 업무 맥락이 사라지는 문제를 해결합니다.

핵심 가치 사용자는 데이터 전환 작업이 아니라 자신의 업무 도움을 받으며, 그 과정 자체가 회사의 AX 데이터로 축적됩니다.

04

기술 스택

프론트엔드

HTML5, CSS3, Vanilla JavaScript SPA, 반응형 리퀴드 글래스 UI, 브라우저 localStorage

백엔드

팀원 Node.js 18+ 내장 HTTP 엔진, 문서 파싱, OKF 온톨로지, 근거 기반 질의·예보·초안·사전점검 REST API

데이터베이스 / 기타

JSON 문서 코퍼스·fixture, 관계 그래프, 업무 ID 기반 컨텍스트 패킷. 운영 단계에서는 문서 저장소·벡터 인덱스·권한 저장소 연결 예정

사용 도구·서비스

Git·GitHub, Node.js·npm, Playwright, Microsoft Edge/Chrome, Codex·Claude. OpenAI API는 로컬 자연어 실험용이며 키와 사내 원문은 저장소에 올리지 않음

프론트엔드와 팀원 백엔드 응답 계약을 연결한 경진대회 통합 데모를 개발하고, 사용자 시연 동선을 테스트하고 있습니다.

완료된 것

  • 업무 수행 기반 ON_메모리 아이디어와 네 가지 업무 요소 확정
  • fixture/live/auto API 어댑터와 유지보수 변환 데이터의 로컬 시연 반영
  • 자연어 입력 + 2주 업무 달력 홈 UI 구현
  • 근거 질의, 업무 작업대, 초안, 제출 전 점검 시연 경로 구현
  • 일정 후보의 확인·되돌리기와 데스크톱·모바일 자동 테스트

진행 중인 것

  • 업무 작업대의 네 가지 요소 요약 강화
  • 메모 대상 변경·취소 등 예외 흐름 회귀 테스트
  • 팀원 백엔드 최신 버전과 live 통합 확인
  • 발표용 대표 시나리오·측정 지표·화면 캡처 정리

06

현재 문제

막힌 점 · 기술·기능

  • 실제 한글·엑셀·전자결재 문서의 안전한 수집·갱신·권한 연동은 아직 프로토타입 범위 밖입니다.
  • 회사 변환 데이터는 현재 로컬 시연용이며, 백엔드 영속 저장과 사용자별 권한 모델이 필요합니다.
  • 유지보수·계약·일반 사무 사용자별로 실제 준비시간이 얼마나 줄어드는지 검증이 더 필요합니다.
  • 근거가 오래되었거나 서로 충돌할 때 최신성·승인 상태를 어떻게 보여줄지 기준을 정교화해야 합니다.

강사 / 수업에 요청

  • HWP·Excel·PDF를 업무 ID에 안전하게 연결하는 인제스트·메타데이터 설계 피드백
  • RAG/온톨로지 답변의 근거성·정확성·재현성을 평가하는 실무 지표
  • 사내 서비스용 인증·권한·감사로그·문서 보존 구조
  • 경진대회에서 기술적 깊이와 사용자 가치를 짧게 증명하는 발표·데모 구성 조언
중요 원칙 근거가 없으면 만들지 않고, AI가 해석한 날짜·기준·결과물은 사용자가 확인한 뒤 업무 메모리로 반영합니다.
07

다음 계획

  1. 진행 중인 예외 흐름 회귀 테스트와 네 가지 업무 요소 작업대 요약을 완료합니다.
  2. 팀원 백엔드 최신 API를 live 모드로 연결하고 동일한 화면이 fixture와 live에서 작동하는지 확인합니다.
  3. 유지보수 기본계획, 공사·용역 착수서류, 사무직 반복업무의 세 가지 대표 시연 시나리오를 완성합니다.
  4. 준비시간, 근거 연결률, 메모-업무 연결 정확도, 재사용 횟수로 경진대회 효과를 측정합니다.
  5. 발표용 배포·녹화·리허설을 완료하고, 이후 실제 사용자 소규모 검증으로 운영 설계를 보완합니다.
