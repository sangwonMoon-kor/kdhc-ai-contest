1. 제품 개요#

1.1 한 줄 정의#

순환근무 조직(한국지역난방공사)의 업무 기억을 사람이 아닌 자리(직무)에 귀속시키는 AI 업무 승계 비서. 발령 첫날 브리핑북이 기다리고, 시즌이 오면 업무가 예보되고, 기안 전 AI가 리스크를 점검하며, 평소에는 내 업무가 자동 분류되어 나만의 매뉴얼로 쌓인다. ⭐

1.2 목표 (우선순위 순)#

  1. 사내 AI 경진대회 우승 — 실서비스 구현이 아니라 직원 공감과 발표 임팩트가 최우선
  2. 심사위원·투표 직원이 데모 3분 안에 "저거 우리 부서에 있으면 쓰겠다"를 느끼게 할 것
  3. 후속 개발 조직(IT부서·외부 파트너)이 문서만으로 착수 가능한 수준의 명세 제공 (기획서 4~7장)

1.3 핵심 가치#

가치설명담당 기능
시간을 아낀다인수인계 적응 단축, 반복 문서 작성 절감기능 ①②
직원을 지킨다감사 지적·반려 사전 예방 — "나를 보호하는 AI"기능 ③
조직에 남긴다문서 적재·태깅이 누적될수록 강해지는 구조기반층 ⭐
나를 위한 도구 ⭐쌓는 게 아니라 쓰다 보면 쌓여 있는 구조 — 도입 동기 선순환기반층(밀착형 업무 매뉴얼)

2. 배경 및 문제 정의#

2.1 문제#

상황현상결과
발령 직후인수인계가 형식적 문서 한 장으로 종료, 과거 문서 소재 파악 불가전임자에게 전화 문의, 적응에 1~2개월
연간 반복업무작년 문서를 못 찾거나 존재를 모름70~80% 동일 문서를 매년 처음부터 작성
기안·결재감사 지적·반려 이력이 개인 기억에만 존재같은 지적·반려 반복, 심리적 부담
평소 ⭐업무 지식 정리는 "인수인계 때 몰아서 하는 일"로 인식정리 유인 부재 → 데이터 공백의 근본 원인

2.2 전략적 포지셔닝 (경진대회 차별화 논리)#

  • 사무 AX 공백: 공사 AX 로드맵(2025.8 발표, 2025.9 AX추진단 출범) 2단계 과제가 전부

설비·현장 중심(안전·열수요예측·경제운전·설비진단) → 전 직원이 매일 하는 기안·인수인계·감사 영역이 빈칸. 본 제안이 사무 AX 첫 사례이자 3단계(AI Agent)로 가는 디딤돌.

  • 선행 사례 대비: 포스코 P-GPT·한화 AIDA는 "검색(수동)", 한국도로공사 AI Draft는 "초안(단건 생성)"에

머묾. 인천공항공사가 2025 공공 AI 왕중왕전 대상을 받은 공식은 "예측 + 화면 표출" → 기능 ②를 알림에서 대시보드로 승격한 근거. SKT "1인 1 AI 에이전트"는 확산 비전(1자리 1메모리)의 근거.

  • 순환근무·감사라는 공기업 고유 맥락을 정면으로 다루는 사례는 국내 미확인.
  • 도입 동기 방어 ⭐: 심사 단골 질문 "직원들이 왜 데이터를 쌓겠나?"에 기반층으로 선제 답변 —

직원은 평소처럼 일할 뿐이고, 그 결과물이 자동 분류되어 본인 매뉴얼(= 미래의 브리핑북)이 된다.

2.3 페르소나#

김민준 과장 — 1월 2일 자로 ○○지사 요금정산팀 발령. 전임자는 이미 본사로 이동. 발표·데모의 전 서사가 이 한 사람의 하루(발령 첫날 → 6개월 뒤 평소 ⭐)를 따라간다. 기능 나열 금지.

3.0 기반층 — 개인별 업무 분류 · 밀착형 업무 매뉴얼 ⭐#

  • 정의: AI가 각자의 결재·메일·회의록을 업무 단위로 자동 분류해 "밀착형 업무 매뉴얼"로

상시 축적하는 층. 직원 입장에서는 매일 쓰는 개인 비서(내 업무 지도·문서 찾기·업무별 요령), 조직 입장에서는 3대 기능의 데이터 기반.

  • 핵심 논리: 브리핑북 = 전임자의 밀착형 매뉴얼. 인수인계 준비가 별도 작업이 아니라

평소 사용의 부산물이 된다.

  • 기술적으로는 기획서 5장 "② 지식 계층(직무-문서 매핑 메타데이터)"의 사용자向 표출 개념 —

아키텍처 신규 요소가 아니라 기존 계층의 재포지셔닝.

3.1 기능 ① 브리핑북 (과거 · 승계) — P1#

  • 발령 이벤트 시 직무 최근 2~3년 문서로 신임자용 요약 리포트 자동 생성.
  • 구성: 직무 개요·핵심 업무 / 연간 업무 달력 / 최근 주요 이슈 3~5건 / 협조처 / 원문 링크.
  • 품질: 모든 항목에 근거 문서 링크 필수, 근거 없으면 "문서 근거 없음" 명시 (환각 방지).
  • 생성: 섹션별 분할 요약 후 통합(맵-리듀스), Gemma 4 중형(256K 컨텍스트). 인사연동은 P2(파일럿은 수동).

3.2 기능 ② 업무 일기예보 (현재 · 예보) — P1#

  • 과거 기안 시점 기반 반복 업무 도래를 달력·예보 카드로 선제 표출, 승인 시 작년 문서 기반 초안 즉시 생성.
  • 예보 로직: 연 1회 이상 · 2년 연속 반복 패턴 탐지 — LLM 불필요, 규칙 기반 배치.

(∴ 데이터 요건: 전자결재 PDF 최소 2년치, 권장 2~3년 — 1년치로는 성립 불가)

  • 대시보드: 월간 달력 + 주간 예보 카드(업무명·작년 기안일·근거 문서 수·D-day) + 부서 뷰/내 자리 뷰.
  • 초안: 전년 문서 구조 분해 → 섹션 단위 생성(장문 일괄 생성 금지). 변경 필요 항목(연도·수치·일정)

하이라이트. 수치·금액은 원문 인용만 허용(생성 금지). "AI 생성 초안 — 검토 필수" 고정 표기.

3.3 기능 ③ 사전 점검 (보호 · 점검) — P2#

  • 기안 제출 전 감사 지적사례·반려 이력·사규와 RAG 대조, 리스크 문구 하이라이트(주의/확인요망) + 근거 제시.
  • 오탐 감수·미탐 최소화의 보수적 설정. "문제 없음" 확정 표현 금지 — 최종 판단은 사람.
  • 감사 데이터 민감성으로 P2 후순위(익명화·유형화 사례집으로 감사부서 협약 후 적재).

경진대회 시연은 각색 샘플로 수행. 발표에서는 이 기능이 감정적 클라이맥스.

3.4 명시적 비범위 (Out of Scope)#

  • 자율 에이전트(도구 선택·자동 기안·자동 결재) — AX 3단계 시점 재검토
  • 외부 클라우드 LLM API — 사내망 원칙 위배
  • 암묵지(대인관계·구두 관행) 승계 — 문서에 없는 지식은 범위 밖임을 UI에 명시

4. 경진대회 데모 요구사항 (1차 산출물 — 구현 완료)#

4.1 원칙#

  • 실서비스가 아닌 시연용 프로토타입. 전 데이터 하드코딩 샘플(실제 부서명·사업명·인명·감사사례

미사용, 각색), LLM 호출 없이 사전 연출. 발표 시 정직 포지션: "화면은 프로토타입, 파이프라인은 사내 인프라(5장 아키텍처)로 구현 가능."

  • 단일 HTML(demo/index.html) + demo/assets/ — 서버·네트워크 불필요, 더블클릭 구동.
  • 해상도 1920×1080(16:9) 고정, 본문 최소 18px(뒷좌석 가독).

4.2 화면 구성 — 7화면 ⭐ (v2.0 기획서는 6화면 기준, 10.3절 갱신 필요)#

화면이름서사 역할상태
S0인트로 — 김 과장의 첫 출근페르소나 몰입. 문서요정이 서류 수집→OCR→자동 분류→대시보드 완성으로 이어지는 스토리보드 컷신 오프닝(스크롤 연동) + 타이틀 전환 ⭐✅
S1브리핑북기능 ① Before/After. "전화로 물어볼 필요가 없다" + "문서 근거 없음" 신뢰 포인트✅
S2업무 일기예보 대시보드기능 ② 메인. 발표 중 실시간 '띵' 알림(A키) → 클릭 → S3✅
S3초안 생성기능 ② 완결. 섹션별 타이핑 연출, 노란 하이라이트 + 체크리스트✅
S4사전 점검기능 ③ 클라이맥스. 리스크 2건 하이라이트 + 근거 패널 + "최종 판단은 담당자"✅
S5 ⭐내 업무 매뉴얼 — 6개월 뒤시점 반전: 받는 사람 → 쌓는 사람. 결재·메일·회의록 4건이 실시간 자동 분류되어 개인 매뉴얼 갱신(버전 bump·문서 수 증가) → "일했을 뿐인데 매뉴얼이 쌓였습니다" → "떠나는 날 이것이 다음 사람의 브리핑북"✅
S6비전 — 1자리 1메모리조직도 점등 + 설비 카르테 한 줄 + 클로징 "당신의 다음 발령지에도, 이게 먼저 가 있습니다"✅ (구 S5)

4.3 S0 상세 (스토리보드 컷신 오프닝 — 기획서 미반영 ⭐)#

스크롤 연동 6컷 이미지 시퀀스(트랙 520vh, 되감기 가능, →로 건너뛰기): 문서요정이 서류 더미에 묻힌 상태(CUT 01) → 벌떡 일어나 정리 자동화 시작(CUT 02) → 필요한 서류를 빠르게 수집(CUT 03) → 모니터가 스캔·OCR 인식(CUT 04) → 계약서·보고서·영수증·신청서 자동 분류(CUT 05) → 정리된 문서와 대시보드 완성(CUT 06) → 컷신이 은은하게 배경화되며 타이틀 "사람은 떠나도, 업무는 남게." 등장(0.96~). 컷 이미지는 demo/assets/storyboard/s0-cut-01.jpg~s0-cut-06.jpg에 저장. 상세 구현·검증 결과는 README 참조.

4.4 S5 상세 (신규 ⭐)#
