# ON_메모리 홈 캘린더 우선 배치 설계

> 상태: 사용자 디자인 승인 완료
>
> 작성일: 2026-07-23
>
> 목적: 홈의 일정과 자연어 입력 순서를 바꾸고, 기존 2주 데이터를 익숙한 Google Calendar식 일정·할 일 표현으로 정돈한다.

## 1. 설계 결론

홈의 정보 우선순위를 다음과 같이 바꾼다.

```text
다가오는 일정
↓ 36px 초점 전환 여백
빠른 자연어 입력
↓
입력 결과·되돌리기
```

사용자는 홈에 들어오자마자 가까운 일정을 먼저 확인하고, 이어서 일정·메모·질문·업무 지시를 자연어로 입력한다. 기존의 `내 업무 일정`은 소유 범위가 중복되고 표현이 딱딱하므로 **`다가오는 일정`**으로 교체한다.

달력은 새 라이브러리를 도입하지 않는다. 이미 검증된 14일 일정 모델과 업무 연결 로직을 유지하면서 Google Calendar의 익숙한 툴바, 날짜 셀, 기간 막대, 오늘 이동 문법을 적용한다. 현재 범위에서는 이 방식이 데이터 회귀 위험과 번들 복잡도를 늘리지 않으면서 가장 빠르게 익숙한 사용성을 제공한다.

## 2. 홈 구조

`vHome()`의 출력 순서를 다음처럼 구성한다.

```text
.home-content
├─ .home-calendar-panel
└─ .home-capture-stack
   ├─ .home-compose
   ├─ .home-feedback
   └─ #homeResult
```

`.home-capture-stack`은 입력과 입력 결과를 한 단위로 묶는다. `.home-content`는 달력과 입력 묶음 사이에 데스크톱 `36px`, 모바일 `24px`의 여백을 둔다. 입력 카드와 피드백 사이의 내부 간격은 기존보다 작게 유지해 둘이 하나의 행동 영역으로 보이게 한다.

자연어 입력의 기능, 파일 첨부, 전송, 결과 표시, 되돌리기 흐름은 변경하지 않는다. 안내 문구도 기존 `지금 어떤 생각을 하시나요?`를 유지한다.

## 3. 캘린더 표현

### 툴바

캘린더 헤더는 다음 요소를 제공한다.

- 제목: `다가오는 일정`
- 현재 2주 범위
- `오늘` 버튼
- 이전 2주 / 다음 2주 버튼
- `전체 일정` 링크

`오늘`은 `homeCalendarOffsetWeeks`를 `0`으로 되돌리고 홈을 다시 렌더링한다. 이미 현재 2주를 보고 있어도 안전하게 동일 상태를 유지한다. 이전·다음 버튼과 전체 일정 링크의 기존 기능은 보존한다.

### 2주 그리드

일요일부터 토요일까지 7열, 두 주를 표시하는 현재 데이터 범위는 그대로 유지한다.

- 날짜는 셀 상단에 놓는다.
- 기준일은 빨간 원으로 표시하고 `aria-current="date"`를 유지한다.
- 기간 업무는 시작일부터 종료일까지 이어지는 가로 막대로 표시한다.
- 마감과 단일 일정은 해당 날짜 안의 짧은 항목으로 표시한다.
- 개인 일정·확인된 메모에는 작은 할 일 표시와 초록 계열을 사용한다.
- 확인 전 후보는 노랑 계열 배경과 점선 경계를 유지한다.
- 업무 종류별 빨강·보라, 개인 일정 초록, 후보 노랑의 기존 의미는 바꾸지 않는다.

이 설계는 Google Calendar처럼 날짜가 있는 업무와 일정을 같은 그리드에서 읽는 익숙한 문법을 차용하지만, ON_메모리의 업무 기간, 직무 메모, 확인 필요 상태는 그대로 보존한다.

## 4. 입력 카드의 시각 위계

입력 카드는 달력 아래의 보조 행동 영역으로 낮춘다.

- 데스크톱 최소 높이는 약 `112px`로 줄인다.
- 과도한 세로 패딩과 전송 버튼의 광원 효과를 한 단계 낮춘다.
- 입력 문구, 첨부 버튼, 전송 버튼의 크기와 키보드 조작 범위는 유지한다.
- 입력 카드 아래의 피드백은 성공, 첨부 오류, 되돌리기 상태를 계속 전달한다.

입력 기능을 숨기거나 축소된 아이콘만 제공하지 않는다. 홈의 핵심 입력 방식이라는 기능적 중요성은 유지하되, 첫 시선은 일정으로 향하게 한다.

## 5. 반응형·접근성

### 데스크톱

- 캘린더와 입력 묶음 사이에 `36px` 간격을 둔다.
- 툴바는 제목, 오늘 이동, 날짜 범위, 이전·다음, 전체 일정 순으로 한 줄에서 읽힌다.
- 14일 그리드는 현재 콘텐츠 폭 안에서 표시한다.

### 태블릿·모바일

- 섹션 사이 간격은 `24px`로 줄인다.
- 툴바는 제목과 주요 이동 조작을 두 줄로 재배치한다.
- 달력은 문서 전체가 아니라 `.home-calendar-scroll` 내부에서만 가로 스크롤한다.
- `오늘`, 이전, 다음 조작은 숨기지 않는다.
- 390px 화면에서 문서 전체 가로 overflow가 없어야 한다.

### 접근성

- 캘린더 영역의 `aria-labelledby`는 새 제목과 연결한다.
- 내부 스크롤 영역의 이름은 `다가오는 2주 일정`으로 바꾼다.
- `오늘`, 이전 2주, 다음 2주 버튼은 고유한 접근성 이름을 가진다.
- 날짜와 일정 항목의 기존 `aria-current`, `aria-label`, 키보드 조작을 유지한다.
- 색만으로 상태를 구분하지 않고 후보의 점선, 할 일 표시, 기존 상태 텍스트를 함께 사용한다.

## 6. 데이터와 컴포넌트 경계

변경은 홈 렌더링과 홈 전용 스타일에 한정한다.

- `home-model.js`: 변경 없음
- `workspace-model.js`: 변경 없음
- `app.js`: DOM 순서, 새 제목, 오늘 버튼과 이벤트 연결만 변경
- `style.css`: 캘린더 툴바, 날짜 셀, 입력 위계, 섹션 간격과 반응형 스타일 변경
- 브라우저 테스트: 새 화면 계약과 상호작용 검증

일정 이벤트 선택 시 업무 작업대 또는 개인 일정 drawer로 이동하는 현재 데이터 흐름은 변경하지 않는다. 일정이 없으면 빈 날짜 셀이 그대로 표시되고, 요약 날짜를 얻지 못하는 기존 오류 처리는 유지한다.

## 7. 검증 기준

브라우저 테스트에서 다음을 확인한다.

1. `다가오는 일정` 제목이 한 번 표시되고 `내 업무 일정`은 표시되지 않는다.
2. 캘린더가 DOM과 시각 순서 모두에서 자연어 입력보다 먼저 온다.
3. 14개의 날짜 셀과 현재 날짜가 유지된다.
4. 다음 2주로 이동한 뒤 `오늘`을 누르면 초기 범위로 돌아온다.
5. 기존 업무·개인 일정·확인 후보를 선택했을 때 연결 동작이 유지된다.
6. 390px에서 문서 전체 가로 overflow가 없고 달력 내부 스크롤만 허용된다.
7. 홈 전용 캡처를 갱신해 달력 우선 위계, 36px 여백, 툴바 가독성을 육안으로 확인한다.

관련 홈 브라우저 테스트와 소스 계약 테스트를 먼저 실패시키고, 구현 후 전체 `npm run test:product-ui`를 실행한다.

## 8. 범위 제외

이번 변경에는 다음을 포함하지 않는다.

- 월간·시간대별 보기
- 드래그 앤 드롭 일정 이동
- 반복 일정
- 외부 Google·Outlook 캘린더 동기화
- FullCalendar 등 새 캘린더 의존성
- 별도 일정 생성 모달의 재설계
- `#schedule` 월간 일정 화면의 구조 변경

이 기능이 필요해지는 시점에는 홈 2주 요약과 전체 일정 화면의 역할을 함께 재검토한다.
