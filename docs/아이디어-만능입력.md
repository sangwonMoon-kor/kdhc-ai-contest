# 아이디어 회의 — 만능 입력 (검색창 · 메모장 통합)

> 2026-07-09 · 참석: 문상원 + Claude
> 상태: **방향 확정, 구현 대기** (다음 라운드 후보 1순위)
> 관련: [demo/app.html](../demo/app.html) (제품 프로토타입 v3) · 팀원 엔진 `creationy/jikmu-memory`

## 1. 발단 — UX를 파다 보니 겹치는 두 입력

v3(3화면: 달력 To-do · 메모장 · 기안)를 만들고 보니, **홈 검색창과 메모장 입력창이 본질적으로 같은 행동**이라는 게 보였다.
사용자 입장에서 원하는 건 하나다:

> **"검색창에 하고 싶은 말을 하면 되는 거 아닐까?"**

- 질문이면 → 답(화면)을 열어주고 — **읽기**
- 진술이면 → 메모로 쌓아주고 — **쓰기**

창구가 하나면 "어디에 입력하지?"라는 선택 자체가 사라진다. 개념적으로도 완성된다:
**묻는 것도, 남기는 것도 같은 행동이고 — 둘 다 자산이 된다.**

## 2. 백엔드 실현 가능성 — 검증 완료 ✅

팀원 엔진(`service/server.js`)을 직접 확인한 결과, **양쪽 경로가 이미 다 있다**:

| 경로 | 엔드포인트 | 상태 |
|---|---|---|
| 읽기(질문) | `POST /api/ask` | ✅ 자체 의도 분류(예보/절차/역할/관계/검색) + LLM 재해석 2-pass 구현됨 |
| 쓰기(메모) | `POST /api/hint/stage` | ✅ 자유 텍스트 → 온톨로지 후보 구조화, 민감정보 가드 포함 |
| 문서 | `POST /api/ingest` (+`/commit`) | ✅ 별도 존재 |

**→ 백엔드 재설계 불필요.** 필요한 건 앞단의 얇은 **"의도 게이트"** 하나: 질문이면 `/api/ask`, 진술이면 `/api/hint/stage`로 분기.
초기엔 클라이언트 규칙(현 PATTERNS 라우팅의 확장)으로 시작하고, 실서비스에서 게이트를 서버로 올리면 된다.

## 3. 질문 vs 메모 — 분류 규칙 (한국어라 신호가 명확)

| 신호 | 질문 (읽기) | 메모 (쓰기) |
|---|---|---|
| 문장부호 | `?` | 없음/마침표 |
| 의문사 | 뭐 · 언제 · 어떻게 · 누구 · 어디 | — |
| 종결어미 | ~해? · ~인가 · ~있어? · ~나요 | ~했음 · ~도착 · ~함 |
| 내용 | — | 사실 나열(날짜 · 금액 · 설비명 · 업체명) |

⚠️ 핵심 사례: **"~해야 해?"는 질문, "~해야 함"은 메모** — 종결어미 하나로 갈린다.

- 규칙만으로 예상 정확도 8~90%
- **애매한 나머지 → 기존 2단계 카드가 받는다**: "🔍 찾아드릴까요 / 📝 적어둘까요" 두 장.
  처음부터 정한 **"최대 2단계" 원칙과 그대로 정합** — 새 UX 부채 없음.

## 4. 오분류 안전장치

- 메모 저장은 **파괴적 동작이 아니다** — 잘못 저장돼도 되돌리기 쉬움
- 저장 확인 줄에 **"질문이었나요? → 검색으로"** 버튼 1개 = 1탭 복구
- 신뢰도가 낮으면 엔진의 **LLM 재해석 계층**(이미 구현됨)으로 위임 가능

## 5. UX 흐름 (확정안)

**홈 검색창 = 만능 입력.**

1. **질문** → 지금처럼 화면 착지 (달력/메모 서재/기안)
2. **진술** → **화면 이동 없이** 검색창 바로 아래 한 줄 확인만:
   `저장됨 · 🔧 정비공사 · ✍️ 할일 추출 → 달력`
   흐름이 안 끊겨 연속으로 메모를 던져 넣을 수 있다 (**퀵 캡처**)
3. **애매** → 2단계 카드 2장 (🔍 / 📝)

역할 재정리:
- **홈** = 유일한 입력 창구 (묻기 + 남기기)
- **메모 화면** = 입력하는 곳이 아니라 **지난 메모를 보고, 분류를 교정하고, 체크리스트로 승격시키는 서재**

## 6. 왜 이게 중요한가 (컨셉 연결)

- 원래 핵심 고민: *"어떤 매개체로, 업무를 늘리지 않고, 스스로 저장하게 할 것인가?"*
- 1차 답 = 검색(읽기 행위의 부산물로 축적) → 2차 진화 = **만능 입력**(쓰기 행위 자체가 축적)
- 검색은 읽기만 만들지만, 메모는 쓰기를 만든다 — **축적 관점에서 더 강한 매개체**
- 캐치프레이즈와 정합: 일한 만큼(묻고 남긴 만큼), 준비됩니다.
