## S0 캐릭터 오프닝 (2026-07-07 추가, zoom-in 전환까지 구현 완료)

**연출**: 마우스 스크롤을 내리는 만큼 진행되는 인수인계 애니메이션. 되감기 가능, `→` 키로 건너뛰기 가능.

1. 인트로(p 0~0.10) — 서류 더미에 파묻힌 캐릭터(포즈0) + 하단 꺼진 모니터
2. 들어올리기(~0.34) — 캐릭터가 빠져나와 파일을 들어올림(포즈1), 실사 파일더미 사진 등장
3. 이동(~0.64) — 파일을 안고 이동(포즈2), 파일더미가 스크롤에 맞춰 점점 작아지다 소멸(0.62)
4. 도착 — 서류를 내밀고(포즈3) 노란 서류철이 모니터 화면 속으로 날아 들어감(0.72~0.80)
5. 점등(0.79) — 모니터 점등("업무 기억 저장 완료 ✓" + 은은한 글로우) + 차임 사운드
6. 화면 확대(0.87~0.98) — 모니터 테두리가 사라지며 화면 레이어만 확대, 주황 빛입자(ai-glow)가
   뷰포트를 가득 채우며 "화면을 뚫고 들어가는" 전환
7. 완료(0.96~) — 장면이 어두워지며 기존 타이틀("사람은 떠나도, 업무는 남게.")·CTA 등장

**구현 위치** (전부 `demo/index.html` 단일 파일):
- CSS: `/* S0 — 스크롤 연동 인수인계 오프닝 */` 블록. zoom-in 전환은 `.s0-stage.zooming` 규칙들
- JS: `s0Update()` 함수 — 스크롤 진행도 p(0~1)에 모든 연출이 묶여 있음. 위 타임라인의 임계값을 여기서 조정.
  스크롤 트랙 길이는 `.s0-track{height:420vh}`
- 확대 기준점: `.s0-mon-wrap`의 `transform-origin:49.9% 38.74%` = `.scr`(화면 유리) 중심.
  `monitor.png`나 `.scr` 좌표를 바꾸면 이 값도 다시 계산해야 함 (scr left+width/2, top+height/2)
- 이미지: `demo/assets/pose0~3.png`(캐릭터, 크로마키 투명 처리본), `pile.png`(실사 파일더미, 크로마키),
  `ai-glow.jpg`(확대 배경 빛입자), `office-bg.jpg`(사무실 배경), `monitor.png`(실사 모니터, 화면 유리
  영역 투명 처리+크롭). 원본은 전부 `demo/assets/src/`에 나노바나나 원본으로 보관
- 로드/새로고침 시 항상 처음부터 재생되도록 스크롤 복원 무시 처리됨 (`initS0`)
- 모니터 화면(점등 연출) 좌표는 `.s0-mon-wrap .scr`에 %로 하드코딩(left 12.01% / top 2.7% / width 75.78% / height 72.07%) — `monitor.png`를 다른 사진으로 교체하면 이 값도 새로 검출해서 맞춰야 함

**참고**:
- `pile.png`에 영어 라벨(ARCHIVE 2022, BUDGETS 등)이 박혀 있음. 축소되는 연출이라 화면에서는 작게
  보여 크게 어색하진 않지만, 거슬리면 나노바나나로 라벨 없는 버전 재생성 후 교체 (크로마키 스크립트
  방식: 테두리 연결 초록 영역만 플러드필 제거 — 내부 초록 바인더 보호)
- `pose3-closeup-src.jpeg`(클로즈업 컷)는 미사용으로 남겨둠 — 기존 포즈3으로 충분하다고 판단
- 발표 해상도 1920×1080 기준으로 검증됨. 확대 최대 배율은 `s0Update()`의 `* 15` (1+zp^2.2*15 ≈ 16배)

**주의**: `index.html` 단독이 아니라 `assets` 폴더가 있어야 캐릭터·배경·모니터가 나옴. 발표장 PC에는 `demo` 폴더째 복사할 것.

## S5 "내 업무 매뉴얼" — 밀착형 업무 매뉴얼 · 개인별 업무 분류 (2026-07-07 추가)

**서사 역할**: S0~S4는 전부 "받는 사람(신임자)" 관점. S5에서 시점을 뒤집어 **"평소에 쌓는 사람"** 관점을 보여줌 —
"인수인계 도구인 줄 알았는데 매일 쓰는 내 비서"라는 반전으로 개인 공감을 만들고, 심사 단골 질문
"직원들이 왜 데이터를 쌓겠나?"에 선제 답변(쌓는 게 아니라 **쓰다 보면 쌓여 있는** 구조).
클라이맥스(S4) 뒤, 비전(S6) 앞에 배치되어 수미상관을 완성.

**연출 타임라인** (화면 진입 시 자동 재생, `runManual()`):
1. 0~2.2초 — 어두운 자막: "발령 6개월 뒤 · 2026년 7월, 요금정산팀 김민준 과장의 평범한 하루"
2. 3초부터 2.3초 간격 — 왼쪽 "오늘 처리한 업무" 피드에 전자결재·회신 메일·회의록 4건이 차례로 유입,
   각 건이 작은 서류 칩으로 오른쪽 "업무 매뉴얼"의 해당 분류로 날아가 꽂힘 →
   분류 카드 하이라이트 + 버전 bump(v2.3→v2.4) + 문서 수 +1 + 틱 사운드
3. 약 13초 — 클로징 오버레이: "따로 정리한 적, 없습니다. 일했을 뿐인데 매뉴얼이 쌓였습니다." →
   "김 과장이 떠나는 날 — 이 매뉴얼이 그대로 다음 사람의 브리핑북이 됩니다." → CTA(비전 보기 / 다시 보기)

**구현 위치** (전부 `demo/index.html`):
- 피드/분류 데이터: JS 상단 `MFEED`(4건: 제목·시각·대상 분류), `MBASE`(분류별 기준 버전·문서 수) — 문구 수정은 여기
- 연출: `runManual()` / `resetManual()` / `stopManual()` / `s5flyTo()`(서류 칩 비행) / `tick()`(짧은 알림음)
- 분류 4종은 S1 브리핑북의 핵심 업무 4개와 대응(요금 정산 / 정산 민원 대응 / 정산시스템 전환 / 계획·보고)
- CSS: `/* S5 내 업무 매뉴얼 */` 블록 (`.fd-*` 피드, `.mn-*` 매뉴얼, `.s5-veil` 자막/클로징)
- S4 하단 버튼이 "6개월 뒤 →"로 변경되어 S5로 연결, S5 클로징 CTA가 S6(비전)으로 연결

## 이 프로젝트를 처음 보는 사람/에이전트를 위한 요약

**배경**: 공기업 순환근무 특성상 담당자가 2~4년마다 바뀌는데, 인수인계가 부실해서
업무 지식이 유실됨. 동시에 연간 반복 업무를 매년 처음부터 재작성하는 비효율이 큼.

**제안 컨셉**: 기반층 + 3대 기능
- **기반층: 개인별 업무 분류** — AI가 각자의 결재·메일·회의록을 업무 단위로 자동 분류해
  "밀착형 업무 매뉴얼"로 상시 축적. 아래 3대 기능이 전부 이 분류 위에서 동작하며,
  직원 입장에서는 "매일 쓰는 내 비서"(데이터 확보 선순환 논리). 데모 S5가 이 층을 보여줌
1. **브리핑북** — 발령 시 신임자에게 과거 문서 기반 요약 리포트 자동 제공 (= 전임자의 밀착형 매뉴얼)
2. **업무 일기예보** — 반복 업무 도래를 달력·예보 카드로 선제 표출 (구 명칭: 시즌 알림)
3. **사전 점검** — 기안 전 과거 감사 지적사례·반려 이력과 대조해 리스크 하이라이트

**차별화 논리** (경진대회 심사 대응):
- 한난은 2025.8 AX 로드맵 발표 + 2025.9 AX추진단 출범 → 로드맵 2단계(안전·열수요예측·
  경제운전·설비진단)가 전부 설비 중심 → **"사무 AX" 공백**을 이 제안이 채운다는 포지셔닝
- 포스코 P-GPT·한화 AIDA는 "검색(수동)"에 머묾, 인천공항공사가 2025 왕중왕전 대상 받은
  이유는 "예측+화면 표출"이었음 → 기능②를 알림에서 대시보드로 승격한 근거
- 발표 감정곡선: 기능①②(효율/빌드업) → 기능③(감사 보호/클라이맥스) → 확장 비전

자세한 배경·리스크·KPI·로드맵은 기획서 docx 참조. 이 리포는 산출물 저장용이며,
기획 배경 전체를 알아야 하면 docx 1~9장을 먼저 읽을 것.

## 데모 사이트 사용법

```
demo/index.html 더블클릭 → 브라우저에서 바로 실행 (서버/네트워크 불필요)
```

- 화면 이동: `←` `→` 방향키, 또는 숫자키 `0`~`6`으로 직행
- `A` 키: 업무 일기예보 화면(S2)에서 알림 배너 재연출
- 딥링크: `index.html#s4` 뒤에 `?auto=1`을 붙이면 사전 점검 화면이 자동 실행됨

화면 구성: S0 인트로 → S1 브리핑북 → S2 업무 일기예보 → S3 초안 → S4 사전 점검 →
**S5 내 업무 매뉴얼(6개월 뒤)** → S6 비전.
⚠️ 기획서 10.3절/10.4절은 S5 추가 전(6화면) 기준이라 화면 번호가 어긋남 — 기획서 반영 작업에서 갱신 예정.
`demo/screenshots/`에 각 화면 렌더링 스크린샷 있음 (S5 추가 전 캡처본 포함, 콘솔 에러 0건 검증).
